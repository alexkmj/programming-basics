Sliding Window

Sliding window is often used for solving overlapping subarray problems in O(n)O(n)O(n) runtime. The main gist of sliding window is memoization. We start with the initial window and we slide the window along the array to find the solution.

Fixed Sliding Window​

A fixed sliding window is a window with a fixed size.

Trivial Example​

Consider the array [1,2,3,4,5,6]\left[1, 2, 3, 4, 5, 6 \right][1,2,3,4,5,6]. Find the subarray of size nnn with that returns the largest sum.

Suppose n=3n = 3n=3. To solve the problem, we initialise a sliding window that matches n=3n = 3n=3 in this case [1,2,3]\left[ 1, 2, 3 \right][1,2,3]. The initial sliding window is a suboptimal solution that sums up to 666. Then, in every step, we deduct the head value from the sum and we add the next value while taking note of the largest subarray.

1  + 2 + 3 =  6
6  - 1 + 4 =  9
9  - 2 + 5 = 12
12 - 3 + 6 = 15

Thus we end up with 15 as the answer.

Dynamically Sized Sliding Window​

A dynamically sized window is window that can be of any size as long as it is not larger than the array we're referencing. We would have to start with a subarray and expand or contract until the condition is matched. This would generally return us a suboptimal solution. Once we reach a suboptimal solution or when we cannot find a solution, we would slide the array around until we find our optimal solution.

Trivial Example​

Consider the array [1,2,3,4,5]\left[1, 2, 3, 4, 5 \right][1,2,3,4,5]. Find the subarray that is greater than or equal to mmm

To solve the problem, we intialise a sliding window. Suppose m=5m = 5m=5, then the initial sliding window is [1]\left[ 1 \right][1]. The intial sliding window is not a solution. Therefore, we continue expanding until we get [1,2,3]=6≥5\left[ 1, 2, 3 \right] = 6 \ge 5[1,2,3]=6≥5. We now have a suboptimal solution. Next we contract our window by removing the head value to get [2,3]=5\left[ 2, 3 \right] = 5[2,3]=5 which is also a suboptimal solution. We try applying the same trick again but notice that [3]=3<5\left[ 3 \right] = 3 < 5[3]=3<5. Therefore instead we move the window to get [2,3,4]=9≥5\left[ 2, 3, 4 \right] = 9 \ge 5[2,3,4]=9≥5. Applying the same trick we eventually get [5]\left[ 5 \right][5] as the subarray.

1                 = 1

674 Longest Contiguous Increasing Subsequence​

In this question, we're asked to find the longest contiguous and increasing subsequence. For example, [1, 3, 5] is the longest continuous increasing subsequence in [1, 3, 5, 4, 7].

1: 1
3: 1, 3
5: 1, 3, 5 
4: 4
7: 4, 7
class Solution {
    public int findLengthOfLCIS(int[] nums) {
        int largest = 0;
        int current = 0;

        for (int index = 0; index < nums.length; index++) {
            if (current == 0) { // initial
                current = 1;
            } else if (nums[index - 1] >= nums[index]) { // reset
                current = 1;
            } else (nums[index - 1] < nums[index]) { // expand
                current = current + 1;
            }

            if (largest < current) {
                largest = current;
            }
        }

        return largest;
    }
}
class Solution:
    def findLengthOfLCIS(self, nums: List[int]) -> int:
        largest = 0
        current = 0

        for index in range(0, len(nums)):
            if (current == 0):
                current = 1
            elif (nums[index - 1] < nums[index]):
                current = current + 1
            else:
                current = 1

            if largest < current:
                largest = current
        
        return largest
